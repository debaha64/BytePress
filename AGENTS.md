# AGENTS

Лёгкая карта для агента в репозитории `BytePress`.

`README.md` — карта для человека: см. `README.md`.

## Назначение BytePress
`BytePress` — файловая фабрика цифровых продуктов с доменной структурой, плановым контуром и журналами фактов.

## Канонические домены и навигация
- `Docs/` — долговременное знание.
- `Docs/Discovery/` — аналитический слой и current-truth интервью.
- `Docs/Technical/` — техническая карта системы и платформенные контракты.
- `Plans/` — roadmap, backlog и утверждённые планы.
- `Logs/` — ADR, изменения и проверки качества.
- `Rules/` — обязательные ограничения.
- `Standards/` — нормативы качества и представления.
- `Tools/` — детерминированные проектные инструменты.

## Рабочий режим
- Humans steer. Agents execute.
- Прямые правки `main` и `develop` запрещены.
- Работа только через task-ветку и PR.
- Перед задачей синхронизация только от `origin/develop`: `git fetch --prune origin` + `git pull --ff-only origin develop`.
- После успешного self-check и публикации ветки агент должен автоматически открыть PR в `develop`, если в среде доступны `gh` и GitHub-аутентификация.
- Если `gh` недоступен или не авторизован, агент не угадывает следующий шаг и выводит готовую команду `gh pr create`, `title` и `body`.
- После merge head-ветка считается закрытой и повторно не используется.
- `release/*` — временная ветка стабилизации только от `develop`.
- В `release/*` не ведётся feature-work; PR из неё направляется только в `main`.
- Если сигнатура дерева репозитория неожиданно усечена, агент останавливается без правок и сообщает о расхождении.
- Commit message, PR title и PR body агент оформляет на английском.
- `branch slug` оформляется на английском в `kebab-case`.
- Язык user-facing взаимодействия определяется параметром профиля `Язык_взаимодействия`.
