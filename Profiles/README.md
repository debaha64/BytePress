# Profiles

`Profiles/` хранит профили режима работы.

## Назначение
- задавать бренд, режим строгости, золотой путь, роли, правила, стандарты, навыки и адаптеры;
- хранить брендовые профили самой фабрики `BytePress`;
- задавать язык пользовательского взаимодействия через поле `Язык_взаимодействия`.

## Границы
В `BytePress` хранятся только брендовые профили. Продуктовые профили создаются в репозитории продукта и обязаны хранить `Код_продукта`. Профиль не хранит журналы, планы, историю изменений и оперативное состояние.

## Карта артефактов
- `Default.md` — брендовый профиль по умолчанию.
- `Speculorg.md` — брендовый профиль Speculorg.
- `README.md` — карта домена профилей.

## Правила
- брендовые профили используют смысловое имя файла;
- `Profiles/*.md`, кроме `Profiles/README.md`, являются hybrid-артефактами профиля;
- внутренний `PROF-*` остаётся каноническим идентификатором профиля;
- в создаваемый продукт из выбранного брендового профиля наследуются только `Брендовый_профиль` и `Язык_взаимодействия`;
- `Код_бренда` в текущем договоре не вводится.
