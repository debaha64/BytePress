# Profiles

Профиль — верхняя единица организации и режима.

## Назначение
В репозитории `BytePress` каталог `Profiles/` хранит только `brand profiles`.

`brand profile` задаёт бренд, режим строгости, активный золотой путь, активные роли, правила, стандарты, навыки и адаптеры для самой фабрики `BytePress`.

В `BytePress` brand profiles сохраняют семантические filenames по бренду и не переводятся на ID в filename.

`Default` трактуется как `brand profile` по умолчанию.

`product profiles` хранятся только в репозитории продукта.

`product profile` обязан хранить `Код_продукта`.

В product repo из выбранного `brand profile` наследуются только `Брендовый_профиль` и `Язык_взаимодействия`.

Внутренние `profile ID` используют 6-значный формат `PROF-<NNNNNN>`.

Параметр `Язык_взаимодействия` хранится внутри профиля и определяет язык user-facing взаимодействия.

`Код_бренда` в текущем контракте не вводится.

## Правило
Профиль не хранит журналы, планы, историю изменений и оперативное состояние.
