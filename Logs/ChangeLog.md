## CHG-000129 — Release archive переупакован по ID-интервалам
ID: CHG-000129
Дата: 2026-05-24
Тип_изменения: Стандарт
Источник: PLAN-000117
Связи: ROAD-000047, BACK-000129, QL-000124
Дата_создания: 2026-05-24
Дата_изменения: 2026-05-24

### Описание
`BACK-000129` закрыт в `PLAN-000117` как исправляющий проход release archive.

Release zip переупакованы по подтверждённым ID-интервалам владельца:
- `0.1.0.zip`: `ROAD-000001` ... `ROAD-000015`, `PLAN-000001` ... `PLAN-000067`;
- `0.2.0.zip`: `ROAD-000016`, `PLAN-000068`;
- `0.3.0.zip`: `ROAD-000017` ... `ROAD-000042`, `PLAN-000069` ... `PLAN-000096`.

Отсутствующих ожидаемых `ROAD` нет. Отсутствующий ожидаемый `PLAN`: `PLAN-000089`, он зафиксирован в `MANIFEST.md` внутри `0.3.0.zip`.

После успешной сборки и проверки zip из текущего дерева удалены `ROAD-000001` ... `ROAD-000042` в `Plans/Archive/Backlog/` и `PLAN-000001` ... `PLAN-000096` в `Plans/Archive/Plans/`. В открытых архивных каталогах остались только `ROAD-000043+` и `PLAN-000097+`. `bp_lint.py` минимально синхронизирован с переносом исторических `ROAD/PLAN` в release zip: hardcoded-сверка `ROAD-000036` / `PLAN-000090` принимает соответствующие файлы внутри `0.3.0.zip`.

### Эффект
Дублирование исторических `ROAD/PLAN` между release zip и текущим деревом устранено без переписывания Git-истории. `ROAD-000047` остаётся `Завершено`, `ROAD-000048` не начат. `ADR-000028` не создан, потому что новое архитектурное решение не принималось: владелец подтвердил распределение release archive по ID-интервалам. `Logs/ReleaseLog.md`, `bp_check.py` и product bootstrap не затронуты.
