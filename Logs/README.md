# Logs

`Logs/` хранит журналы подтверждённых фактов.

## Назначение
- фиксировать факты решений, изменений, качества, выпусков и сопровождения;
- отделять историю от текущего плана и временного исполнения;
- давать проверяемый след закрытия проходов.

## Границы
Журнал не планирует работу, не хранит прогнозы и не заменяет `Plans/*`. Старые записи не переписываются без отдельного основания; текущий проход добавляет только новые факты. Если обнаружен доказанный пропуск решения, ретро-ADR добавляется новой записью с явной связью с исходным проходом.

## Карта артефактов
- `ADRlog.md` — значимые архитектурные, процессные и продуктово-договорные решения.
- `ChangeLog.md` — значимые изменения.
- `QualityLog.md` — проверки и результаты контроля.
- `ReleaseLog.md` — подтверждённые выпуски и передачи.
- `SupportLog.md` — факты сопровождения.

## Правила
- журнальные файлы остаются singleton-артефактами со смысловыми именами;
- записи журналов идут хронологически от старых к новым;
- новые записи добавляются в конец соответствующего журнала;
- верхняя навигация допустима только как служебный блок с кратким статусом, а не как полный дубль журнала;
- serial `ID` получают внутренние записи журналов там, где журнал ведёт serial fact registry;
- ссылка на конкретную запись идёт по её внутреннему `ID`, путь к файлу добавляется только для навигации;
- содержательный завершённый проход не должен оставлять доказанный пропуск в `ChangeLog.md` или `QualityLog.md`.
- значимое архитектурное, процессное или продуктово-договорное решение не должно оставаться только в `ChangeLog.md`, `QualityLog.md`, плане или техническом документе без ADR.
- запись о закрытии `ROAD/BACK/PLAN` не должна расходиться с фактическим состоянием `Plans/*`.
