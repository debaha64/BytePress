# Memory

`Memory/` хранит каркас домена долговременной памяти будущих версий `BytePress`.

## Назначение
- описать, что в будущем может считаться устойчивой памятью системы;
- развести память, журналы, документацию и оперативное состояние;
- подготовить границы и интерфейсы для перехода от файлового режима к более сложным механизмам хранения.

## Статус `0.1.0`
Только границы, модель, интерфейсы и заглушки. Реальная память не реализуется.

## Состав домена
- `Model.md` — модель памяти;
- `Boundaries.md` — границы и запреты;
- `Interfaces.md` — будущие интерфейсы;
- `Registry.md` — реестр будущих типов памяти.
