# Memory

`Memory/` хранит каркас будущего домена долговременной памяти.

## Назначение
- описать возможную устойчивую память будущих версий;
- развести память, журналы, документы и оперативное состояние;
- подготовить границы и интерфейсы для более сложных механизмов хранения.

## Границы
В текущей версии `Memory/` не является источником истины и не подменяет `Docs/`, `Logs/` или `Runtime/`. Реальная память не реализована.

## Карта артефактов
- `Model.md` — модель будущей памяти.
- `Boundaries.md` — границы и запреты.
- `Interfaces.md` — будущие интерфейсы.
- `Registry.md` — реестр будущих типов памяти.

## Куда идти дальше
- техническая модель: `Docs/Technical/Model.md`;
- журналы фактов: `Logs/README.md`;
- временный контекст: `Runtime/README.md`.
