# Policy

## Назначение
Этот документ задаёт общую политику подключения моделей и движков к `BytePress`.

## Правила
1. Адаптер активируется только через профиль.
2. Адаптер не меняет золотой путь и не ослабляет правила автоматически.
3. Адаптер может добавлять только операционные параметры: среду запуска, допустимые роли, способы вызова, режимы ограничений.
4. Канонический режим `v1` использует одного активного исполнителя за раз.
5. Резервные адаптеры могут использоваться для review, critique или research, но не считаются основным контуром исполнения.
6. Локальные движки допускаются только через отдельный адаптер `Local`, без внедрения их частных настроек в системные домены.

## Границы
- сведения о бренде остаются в `Profiles/`;
- правила и стандарты остаются в `Rules/` и `Standards/`;
- процедурные действия остаются в `Skills/`;
- детерминированные операции остаются в `Tools/`.
