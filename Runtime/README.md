# Runtime

`Runtime/` хранит временный контекст текущего исполнения.

## Назначение
- поддерживать рабочий контекст задачи;
- хранить временные заметки исполнения;
- помогать передаче смены;
- не подменять документы, планы и журналы.

## Границы
`Runtime/` не является источником истины. После завершения задачи он очищается или остаётся только с временными файлами, которые не заменяют `Docs/*`, `Plans/*` и `Logs/*`.

## Карта артефактов
- `Context.md` — текущий рабочий контекст.
- `Task.md` — оперативная постановка задачи.
- `Session_Log.md` — временный след сессии.
- `Handover.md` — передача смены.
- `README.md` — карта временного домена.

## Куда идти дальше
- канонический план: `Plans/README.md`;
- факты результата: `Logs/README.md`;
- границы временного состояния: `Rules/Runtime_Is_Temporary.md`.
