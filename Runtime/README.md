# Runtime

`Runtime/` — оперативная среда текущего исполнения.

## Назначение
- хранить текущий контекст задачи;
- хранить оперативный черновик исполнения;
- поддерживать передачу смены;
- не подменять `Docs/`, `Plans/` и `Logs/`.

## Политика
После завершения задачи `Runtime/` очищается. Необходимые следы остаются в `Plans/` и `Logs/`.

## Статус `0.1.0`
Канонический временный домен исполнения. Используется только во время активного task-pass.

## Участие в lifecycle
`Runtime/` участвует между активным `Plan` и фиксацией результата в `Logs/*`, когда нужен рабочий контекст или передача смены.

## Границы
- `Runtime/` не является источником истины;
- `Runtime/` не подменяет `Docs/`, `Plans/` и `Logs/`.
