# Workflow

## RULE-000001 — Approval strictness
ID: RULE-000001
Название: Approval strictness
Статус: Активно
Связи: PROF-000001, PROF-000002, Pipeline/Phase_Gates.md
Источник: Базовый регламент BytePress
Дата_создания: 2026-03-09
Дата_изменения: 2026-04-30
Обязательность: Обязательно
Связанные_договоры:
- Pipeline/Phase_Gates.md

### Назначение
Режим строгости задаётся профилем и определяет набор обязательных ручных утверждений между фазами конвейера.

### Нарушение
Переход между фазами без требуемого для активного профиля утверждения.

### Проверка
Проверка активного профиля, фазовых ворот и связанного плана.

## RULE-000008 — Plans require approved backlog
ID: RULE-000008
Название: Plans require approved backlog
Статус: Активно
Связи: ADR-000003, Pipeline/Phase_Gates.md
Источник: Базовый регламент BytePress
Дата_создания: 2026-03-10
Дата_изменения: 2026-04-30
Обязательность: Обязательно

### Назначение
`Plan` не создаётся из пустоты. Для плана должен существовать backlog-элемент, связанные требования и, при необходимости, связанное архитектурное решение.

### Нарушение
План появляется без основания, без связей на backlog и без подтверждённого входа в работу.

### Проверка
Проверка поля `Основание`, связей на `Backlog` и связанного пакета документации.

## RULE-000017 — Русский пользовательский вывод агента
ID: RULE-000017
Название: Русский пользовательский вывод агента
Статус: Активно
Связи: AGENTS.md, Pipeline/Workflows.md
Источник: Исправляющий проход 2026-05-05
Дата_создания: 2026-05-05
Дата_изменения: 2026-05-05
Обязательность: Обязательно

### Назначение
Все записи агента в артефакты `BytePress` и весь видимый пользователю текст ответа агента должны быть на русском языке.

Английский допускается только для имён собственных, путей, команд, веток, ID и технически неизбежных мест.

### Нарушение
Агент пишет пользователю или в артефакты `BytePress` англоязычный текст там, где нет технической необходимости.

### Проверка
Проверка стартовой карты агента, рабочего маршрута и финального отчёта прохода.
