# Naming

## RULE-000015 — Semantic path and file naming
ID: RULE-000015
Название: Semantic path and file naming
Статус: Принято
Связи: ADR-000024, RULE-000007, RULE-000010
Источник: Запрос владельца от 2026-04-30
Дата_создания: 2026-04-30
Дата_изменения: 2026-04-30
Обязательность: Обязательно

### Правило
Путь задаёт домен, а имя файла задаёт предмет договора внутри этого домена.

### Обязательные условия
- имя файла не должно быть длинным предложением;
- одинаковые сущности называются одинаково во всех активных ссылках;
- файл с правилом, договором или картой называется по области договора, а не по полной формулировке условия;
- исторические файлы можно не переименовывать без отдельной причины;
- при переименовании active artifact обновляются прямые ссылки, checks и generated artifacts.

### Нарушение
Файл получает имя вроде полного тезиса правила, одна и та же сущность называется по-разному в соседних owner-documents, либо путь и имя файла вместе дублируют одно и то же значение.

### Проверка
Проверка карты домена, прямых ссылок, обязательных списков в tools и соответствия `Rules/README.md` фактическим файлам `Rules/*`.
