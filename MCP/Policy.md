# Policy

## Базовые правила
1. Подключение должно быть явным и регистрируемым.
2. Подключение не подменяет файловый источник истины `BytePress`.
3. Подключение не должно обходить профили, правила и стандарты.
4. Любое подключение должно иметь определённую область доступа и назначение.
5. Подключение должно использоваться через управляемые интерфейсы, а не как произвольный внешний обход доменной модели.
6. Product-side scripts и tooling могут использовать только controlled handoff к `MCP/*` через repo-native `BytePress` tools.
7. Реальные внешние вызовы, секреты, токены и network-dependent runtime logic остаются вне текущего active contour.

## Допустимые причины подключения
- поиск и чтение внешних данных;
- работа с issue trackers и репозиториями;
- доступ к логам и журналам продукта;
- интеграция с контейнерным контуром продукта через ограниченные точки входа.
- deterministic local smoke handoff generated product repo к declared connector contour без открытия внешней сети.

## Недопустимые применения
- хранить в MCP основной источник истины;
- держать там оперативный контекст задачи;
- подменять `Tools/` и проектные скрипты произвольными вызовами внешней среды.
- вызывать внешние сервисы напрямую из generated product repo в обход repo-native tooling.
