# MCP

`MCP/` хранит каркас подключений и политики интеграции `BytePress`.

## Назначение
- зафиксировать, зачем системе нужен слой подключений;
- развести файловый источник истины и внешние инструменты;
- подготовить реестр допустимых подключений и границы их использования.

## Статус `0.1.0`
Только политика, интерфейсы и реестр. Реальные подключения не являются частью базового выпуска `0.1.0`.

## Участие в lifecycle
`MCP/` в `0.1.0` участвует только когда изменение касается политики подключений и интеграционного контура. В обычном task-pass домен не становится активным источником данных.

## Границы
- `MCP/` не является источником истины;
- `MCP/` не подменяет файловый репозиторий, `Adapters/` и knowledge-domains.

## Состав домена
- `Policy.md` — общая политика подключений;
- `Interfaces.md` — будущая модель интерфейсов;
- `Registry.md` — реестр кандидатов и статусов подключения.
