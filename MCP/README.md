# MCP

`MCP/` хранит контур подключений и политики интеграции `BytePress`.

## Назначение
- объяснить, зачем системе нужен слой подключений;
- развести файловый источник истины и внешние инструменты;
- держать реестр допустимых классов подключений и границы их использования.

## Границы
`MCP/` не является источником истины, не подменяет файловый репозиторий, `Adapters/` и домены знания. Bootstrap не переносит `MCP/*` в создаваемый продукт.

## Карта артефактов
- `Policy.md` — политика подключений.
- `Interfaces.md` — модель контролируемой передачи во внешний контур.
- `Registry.md` — реестр кандидатов и статусов подключений.
