# Pipeline

`Pipeline/` хранит канон движения работы по фазам.

## Назначение
- описать фазы, входы, выходы и передачи процесса;
- удерживать границы между процессом, планированием, исполнением и журналами;
- задавать процессные гейты без подмены документов-владельцев в других доменах.

## Границы
`Pipeline/` не владеет текущим `ROAD/BACK/PLAN`, не хранит факты изменений и не заменяет технические договоры. Эти состояния принадлежат `Plans/*`, `Logs/*` и `Docs/Technical/*`.

## Карта артефактов
- `README.md` — карта процессного домена.
- `Phases.md` — фазы и порядок движения работы.
- `Phase_Gates.md` — условия переходов и процессные гейты.
- `Inputs_Outputs.md` — входы, выходы и передачи между фазами.
- `Artifacts.md` — участие артефактов в процессе.
- `Golden_Path.md` — основной рабочий маршрут.
- `Methods/` — описания допустимых методных вариантов.

## Куда идти дальше
- текущий плановый контур: `Plans/README.md`;
- техническая роль процесса: `Docs/Technical/Pipeline.md`;
