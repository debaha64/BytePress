# Pipeline

`Pipeline/` хранит формальную модель движения работы в BytePress.

## Назначение
- описывать фазы, переходы, входы и выходы изменения;
- задавать общий порядок движения работы без хранения фактического статуса задачи.

## Статус `0.1.0`
Канонический домен модели процесса. Используется как техническая рамка, а не как отдельный источник текущих данных.

## Участие в lifecycle
`Pipeline/` участвует всегда, когда нужно понять порядок фаз и ворот изменения, но не хранит текущую истину, статус выполнения или факт завершения.

## Границы
- `Pipeline/` не является источником истины;
- `Pipeline/` не подменяет `Plans/`, `Logs/` и `Docs/Technical/Artifact_Lifecycle.md`.
