# BytePress

BytePress — файловая фабрика цифровых продуктов: доменная база знаний, плановый контур и журналы фактов в одном репозитории.

`README.md` — карта для человека. `AGENTS.md` — карта для агента.

## Назначение
- держать систему и продукт в проверяемом файловом контуре;
- формировать устойчивый контекст вокруг продукта;
- удерживать агента в управляемом рабочем цикле;
- снижать недетерминированность разработки;
- вести работу через `Roadmap -> Backlog -> Plan`;
- фиксировать решения, изменения и проверки в журналах;
- поддерживать воспроизводимую работу через Git и PR.

## Границы
`README.md` не хранит договоры доменов и не заменяет документы-владельцы. При споре приоритет у активного документа соответствующего домена.

## Карта доменов
- `Docs/` — долговременное знание.
- `Docs/Discovery/` — аналитическая истина и интервью.
- `Docs/Product/` — продуктовая рамка и поставка.
- `Docs/Technical/` — системные договоры, границы, жизненный цикл и проверки.
- `Docs/Terms/` — словарь и правила терминологии.
- `Plans/` — дорожная карта, реестр работ, текущий план и архив планирования.
- `Logs/` — факты решений, изменений, качества, выпусков и сопровождения.
- `Rules/` — обязательные ограничения.
- `Pipeline/` — фазы, рабочие потоки, гейты, уровни проверок, журнальное закрытие и PR-маршрут.
- `Tools/` — детерминированные инструменты и проверки.
- `Profiles/` — брендовые профили для bootstrap.
- `Schemas/`, `Templates/` — формальные схемы и шаблоны документов.
