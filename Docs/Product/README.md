# Product

`Docs/Product/` хранит продуктовую рамку `BytePress` или создаваемого продукта.

## Назначение
- описать прикладную ценность, требования и поставку;
- удерживать продуктовый смысл отдельно от технических договоров;
- дать минимальный стартовый слой для создаваемого продукта.

## Границы
`Docs/Product/` не хранит технический договор начального развёртывания, текущий план, журнальные факты или аналитическую истину. Для них используются `Docs/Technical/*`, `Plans/*`, `Logs/*` и `Docs/Discovery/*`.

## Карта артефактов
- `README.md` — карта продуктового домена.
- `JTBD.md` — задачи пользователя и ожидаемый результат.
- `PRD.md` — продуктовые требования.
- `Delivery.md` — модель поставки и передачи.

## Куда идти дальше
- текущая аналитическая истина: `../Discovery/README.md`;
- технический слой: `../Technical/README.md`;
- плановый контур: `../../Plans/README.md`;
- журналы: `../../Logs/README.md`.
