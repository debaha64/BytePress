# User

`Docs/User/` хранит пользовательский вход в `BytePress`.

## Назначение
- объяснить человеку, как входить в систему;
- описать режим взаимодействия человека, агента и репозитория;
- зафиксировать первый маршрут старта и типовые сценарии.

## Границы
`Docs/User/` не является техническим договором, плановым контуром или журналом фактов. Он ссылается на владельцев смысла и остаётся кратким пользовательским слоем.

## Карта артефактов
- `README.md` — карта пользовательского домена.
- `Operating_Mode.md` — режим работы человека с агентом и репозиторием.
- `First_Start.md` — первый маршрут после получения репозитория.
- `Setup_Guide.md` — пользовательская подготовка локальной среды.
- `Pass_Request.md` — как формулировать запрос на проход для агента.
- `Usage_Scenarios.md` — базовые сценарии использования.
