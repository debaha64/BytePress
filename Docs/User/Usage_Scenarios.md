# Сценарии использования

## Назначение
Этот документ фиксирует минимальные базовые сценарии использования `BytePress` как системы.

Сценарии здесь остаются короткими и направляют к документам-владельцам вместо того, чтобы дублировать их.

## Сценарий 1. Войти в существующий репозиторий BytePress
- Прочитать `../../README.md` как карту системы.
- Проверить режим работы человека через `Operating_Mode.md`.
- Если среда не готова, пройти `Setup_Guide.md`.
- Из корня репозитория запустить `python3 Tools/bp_lint.py --repo .`, чтобы убедиться, что базовая структура валидна.

## Сценарий 2. Запустить новый проход с агентом
- Найти текущий этап/задачу/проход в `../../Plans/Roadmap.md`, `../../Plans/Backlog.md` и текущем `Plan`.
- Сформулировать запрос на проход через `Pass_Request.md` и договоры репозитория.
- Направить агента на задачу, опираясь на договоры репозитория, а не на ручной пересказ всего рабочего цикла.
- Проверить результат по проверкам, управленческой сверке и маршруту PR, которые агент обязан пройти по `../../AGENTS.md`.

## Сценарий 3. Использовать BytePress как файловую систему знаний и планирования
- Смотреть `Docs/*` как долговременное знание по доменам.
- Смотреть `Plans/*` как канонический плановый слой.
- Смотреть `Logs/*` как слой фактов, изменений и доказательств качества.
- Смотреть `Tools/*` только как исполняемую поддержку поверх уже утверждённых договоров.

## Сценарий 4. Понять, куда идти по типу вопроса
- настройка, среда, зависимости: `Setup_Guide.md`;
- рабочий канон агента: `../../AGENTS.md`;
- техническая модель и контракты: `../Technical/README.md`;
- плановое состояние и текущая работа: `../../Plans/README.md`;
- продуктовая рамка: `../Product/README.md`.

## Граница сценариев
- Эти сценарии не являются полным пользовательским руководством.
- Они задают только минимальный первый рабочий слой для человека.
- Расширение пользовательского слоя требует отдельного прохода внутри текущего этапа пользовательской документации, а не неявного роста этого документа.
