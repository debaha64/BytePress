# First Start

## Назначение
Этот документ задаёт первый маршрут входа в `BytePress` для человека-пользователя.

Он не дублирует `Setup_Guide.md`, а помогает быстро понять, какой owner-document открыть следующим.

## Первый маршрут
1. Открыть [../../README.md](../../README.md) и понять, что такое `BytePress` и какие у него домены.
2. Открыть [Operating_Mode.md](Operating_Mode.md) и зафиксировать human steering mode.
3. Открыть [Setup_Guide.md](Setup_Guide.md), если среда ещё не подготовлена или нужна проверка локального контура.
4. Открыть [../../Plans/README.md](../../Plans/README.md), чтобы увидеть, где живут `Roadmap`, `Backlog` и текущий `Plan`.
5. Из корня репозитория запустить базовую проверку структуры через `python3 Tools/bp_lint.py --repo .`.

## Как понять, что старт уже достаточный
- человек понимает назначение системы и её доменную карту;
- рабочая среда проверена по пользовательскому setup-документу;
- плановый слой читается без двусмысленности;
- понятно, где искать operating canon агента и где искать технические контракты.

## Чего этот маршрут не делает
- не заменяет настройку среды из [Setup_Guide.md](Setup_Guide.md);
- не объясняет технические контракты вместо [../Technical/README.md](../Technical/README.md);
- не создаёт новое плановое состояние или task-сценарий вместо [../../Plans/README.md](../../Plans/README.md).

## Куда идти после старта
- если нужен новый pass или статус работы: [../../Plans/README.md](../../Plans/README.md);
- если нужен agent-assisted execution: [../../AGENTS.md](../../AGENTS.md);
- если нужно корректно сформулировать pass: [Pass_Request.md](Pass_Request.md);
- если нужны базовые паттерны использования: [Usage_Scenarios.md](Usage_Scenarios.md).
