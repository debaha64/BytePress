# System_Invariants

## Инварианты доменной карты
1. `Docs/` не хранит оперативное состояние.
2. `Runtime/` не хранит долговременное знание.
3. `Plans/` содержит только утверждённые планы и плановый контур продукта.
4. `Logs/` содержит только факты, а не планы или временные заметки.

## Инварианты управления
5. Режим строгости задаётся профилем и должен быть параметризуемым.
6. Правило не равно стандарту.
7. Навык не равен инструменту.
8. Схема не равна шаблону.
9. Адаптер модели не равен знанию системы.

## Инварианты терминологии
10. Новый термин нельзя вводить без проверки на конфликт и без решения.
11. Определение термина хранится только в карточке термина.
12. Ссылки на ключевые сущности в системе даются по ID.

## Инварианты планирования и журналов
13. `Roadmap -> Backlog -> Plan` — обязательный плановый контур.
14. `Plan` не создаётся без backlog-элемента и связанных оснований.
15. История статусов не размазывается по артефактам, а фиксируется журналами значимых изменений.
16. Значимое утверждённое изменение должно попасть в `ChangeLog`.
17. Архитектурно значимое решение должно попасть в `ADRlog`.

## Инварианты исполнения
18. После завершения задачи `Runtime/` очищается.
19. Временное исполнение не становится вторым источником истины.
20. Контейнерный контур продукта не считается контуром агента.
