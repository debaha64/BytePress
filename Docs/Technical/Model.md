# Model

## Базовая иерархия
- Профиль — верхняя единица организации и режима.
- Продукт — единица поставки.
- Репозиторий — физический носитель системы или продукта.
- Конвейерная работа — единица изменения.
- Артефакт — единица данных.

## Сущности `v1`

### Профиль
Профиль задаёт:
- бренд;
- политику строгости;
- активный золотой путь;
- активные роли;
- активные правила и стандарты.

Профили делятся на два типа:
- `brand profile` хранится в репозитории `BytePress` и задаёт режим бренда для самой фабрики;
- `product profile` хранится в репозитории продукта и задаёт режим конкретного продукта.

`Default` трактуется как `brand profile` по умолчанию.

`product profile` дополнительно хранит `Код_продукта`.

`Код_бренда` в текущем контракте не вводится.

### Продукт
Продукт является поставляемой сущностью. Продукт использует `BytePress` как фабрику, но не сливается с самой системой.

### Репозиторий
Репозиторий — физический контейнер файлов. Репозиторий может хранить либо саму систему `BytePress`, либо отдельный продукт.

### Конвейерная работа
Конвейерная работа начинается с обсуждения, проходит через аналитические и плановые фазы, затем через реализацию и проверку, и завершается выпуском, передачей и поддержкой.

### Артефакт
Артефакт — любая типизированная сущность системы: термин, план, запись журнала, роль, правило, стандарт, элемент `Roadmap`, элемент `Backlog`.

## Отношения сущностей
- профиль активирует роли, правила, стандарты и золотой путь;
- `BytePress` хранит только `brand profile`, а продуктовый репозиторий хранит только `product profile`;
- пакет основания документации порождает `Roadmap` и `Backlog`;
- элемент `Backlog` плюс связанные требования и решения порождает `Plan`;
- `Plan` становится основанием для реализации;
- факты реализации переходят в журналы;
- термины и стандарты влияют на все текстовые артефакты.

## Активное и пассивное состояние

### Активное
- `Runtime/`
- текущая запись в `Plan`
- текущая фаза конвейера

### Пассивное
- `Docs/`
- `Logs/`
- завершённые элементы `Backlog`
- завершённые `Plan`

## Состояния сущностей
Короткий набор статусов используется для `Backlog` и `Plan`:
- `Черновик`
- `Готово_к_утверждению`
- `Утверждено`
- `В_работе`
- `Завершено`
- `Отменено`
