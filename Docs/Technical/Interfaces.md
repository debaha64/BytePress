# Interfaces

## Внутренние интерфейсы доменов

### `Docs/` -> `Plans/`
Знание системы и продукта становится основанием для `Roadmap`, `Backlog` и `Plan`.

### `Pipeline/` -> `Plans/`
Модель фаз определяет, когда допускается создание или обновление плановых артефактов.

### `Plans/` -> `Runtime/`
Утверждённый `Plan` становится входом текущего исполнения, но не копируется в `Runtime/` как новый источник истины.

### `Runtime/` -> `Logs/`
Подтверждённые факты исполнения переходят в журналы.

### `Profiles/` -> `Pipeline/`, `Roles/`, `Rules/`, `Standards/`
Профиль включает режим строгости и активирует набор допустимого поведения.

### `Roles/` -> `Skills/`, `Rules/`, `Standards/`
Роль собирает допустимое поведение через ссылки на навыки, правила и стандарты.

### `Schemas/` -> `Templates/` -> документы
Схема задаёт обязательные поля, шаблон даёт читаемое представление, а документ реализует конкретный экземпляр артефакта.

### `Tools/` -> все домены
Инструменты проверяют, генерируют или нормализуют артефакты без подмены доменных смыслов.

## Внешние интерфейсы системы

### Рабочая среда
`BytePress` рассчитан на Linux или WSL2, а не на Windows-native контур агента.

### Контур моделей
Адаптеры моделей вынесены в `Adapters/` и не определяют архитектуру самой системы.

### Контур продуктов
`BytePress` управляет фабрикой и документацией, но продукт остаётся отдельной поставляемой сущностью.

### Контур контейнеров
Контейнеры продуктов рассматриваются как часть продукта, а не как часть среды агента.

## Формат ссылок
Внутренние связи между сущностями задаются по ID. При необходимости ID может сопровождаться ссылкой на файл или раздел.

## Связи
- `RULE-000003`
- `RULE-000007`
- `RULE-000009`
- `STD-000002`


### `Adapters/` -> `Profiles/`, `Roles/`, `Skills/`
Адаптер связывает конкретную модель с уже утверждённым режимом и не может менять доменную модель системы.

### `Memory/` -> `Logs/`, `Docs/`
Память будущих версий может извлекать устойчивое знание только из уже подтверждённых документов и журналов.

### `MCP/` -> `Adapters/`, `Tools/`
Подключения будущих версий должны использоваться только через управляемые интерфейсы и не заменять проектные инструменты и скрипты.
