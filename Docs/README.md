# Docs

`Docs/` хранит долговременное знание `BytePress` и создаваемого продукта.

## Назначение
- хранить устойчивые документы, которые должны переживать один рабочий проход;
- разделять пользовательский, аналитический, продуктовый, технический и терминологический слои;
- не смешивать знание с временным исполнением, планированием и журналами фактов.

## Границы
`Docs/` не хранит оперативные заметки, текущие планы, журнальные факты и временный контекст. Для них используются `Plans/*`, `Logs/*` и ignored tool-output paths.

## Карта артефактов
- `Docs/User/` — пользовательский вход и сценарии работы.
- `Docs/Discovery/` — текущая аналитическая истина, интервью, обсуждения, исследования и требования.
- `Docs/Product/` — продуктовая рамка, ценность, требования и поставка.
- `Docs/Technical/` — технические договоры, границы, жизненный цикл, проверки и начальное развёртывание.
- `Docs/Terms/` — словарь терминов и правила изменения терминологии.
