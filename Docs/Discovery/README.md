# Discovery

`Docs/Discovery/` хранит долговременное аналитическое знание до перехода в планирование и исполнение.

## Назначение
- зафиксировать текущую аналитическую истину;
- собрать вопросы, ответы, обсуждения, исследования и требования;
- отделить подтверждённые ответы от заготовок начального развёртывания.

## Границы
`Docs/Discovery/` не хранит продуктовую реализацию, технический договор, плановое состояние или журнальные факты. Первый старт созданного продукта остаётся только аналитическим, пока `Interview.md` не подтверждён пользователем.

Созданный начальным развёртыванием продукт стартует с `Статус_текущей_истины: Не_подтверждена`; до подтверждения пользователем создаваемый продукт остаётся только в аналитическом контуре. Агент не предлагает дополнительный функциональный состав первой версии как пример, если пользователь его не назвал.

## Карта артефактов
- `README.md` — карта аналитического домена.
- `Discussion.md` — предмет обсуждения, вопросы и границы.
- `Interview.md` — владелец текущей аналитической истины.
- `Research.md` — проверенные факты, ограничения и альтернативы.
- `Requirements.md` — обязательные требования и границы следующего этапа.

## Договор интервью
- единственный владелец протокола интервью — `Interview.md`;
- вопросы первого прохода собираются по классам `Результат`, `Граница`, `Вне границы`, `Ограничение`, `Стек и зависимости`, `Владение`, `Блокеры`;
- блокирующий вопрос задаётся сразу;
- неблокирующий вопрос переносится в следующую фазу;
- заготовки начального развёртывания не считаются подтверждённой текущей истиной.
