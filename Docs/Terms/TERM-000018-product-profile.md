# Профиль продукта

ID: TERM-000018
Термин: Профиль продукта
Статус: Принят
Связи: TERM-000001, TERM-000002, TERM-000017
Источник: Сверка продуктового слоя и профилей BytePress
Дата_создания: 2026-03-26
Дата_изменения: 2026-05-11
Решение_по_конфликту: Принять как профильный выбор пакетов каркаса без обязательного домена `Profiles/*` внутри продукта

## Определение
Профильный выбор пакетов создаваемого продукта, который определяет состав каркаса при начальном развёртывании: core `Docs/*`, `Plans/*`, `Logs/*`, `Pipeline/*`, `Tools/*`, bounded `Templates/*` и bounded `Schemas/*`.

Профиль продукта не обязан храниться в product repo как `Profiles/Product.md`. В текущей модели `Profiles/*` не materialize в продукт.

## Синонимы
- Product Profile

## Связанные_термины
- TERM-000001
- TERM-000002
- TERM-000017

## Паспорт созданного каркаса
Локальный живой паспорт продукта хранится в `Docs/Product/Product_Passport.md`.

Минимальные поля паспорта:
- `Название`
- `Код_продукта`
- `Брендовый_профиль`
- `Язык_взаимодействия`
- `Пакеты_каркаса`
- `Тип_продукта`
- `Основной_запуск`
- `Подтверждённый_стек`
- `Предметные_пакеты`
- `Ограничения_среды`

На старте предметные поля остаются `Не_подтверждено`. После первого предметного прохода паспорт фиксирует подтверждённые значения без возвращения домена `Profiles/*` в продукт и без создания отдельного governance-layer.
