# Terms

`Docs/Terms/` хранит терминологический слой `BytePress`.

## Назначение
- удерживать терминологию как нормативный язык BytePress;
- держать одно значение для одного термина;
- оформлять новые термины отдельными карточками;
- отделять словарь `BytePress` от словарей создаваемых продуктов.

## Границы
Определение термина хранится только в карточке термина. `README.md` и `Base_Terms.md` являются навигацией и индексом, а не вторым местом определения.

## Карта артефактов
- `README.md` — карта терминологического домена.
- `Base_Terms.md` — индекс базовых терминов.
- `Term_Change_Policy.md` — порядок изменения словаря.
- `TERM-<NNNNNN>-<slug>.md` — карточка термина.

## Правила
- маршрут терминологии: `AGENTS.md` -> `Rules/Terms.md` -> `Docs/Terms/*`;
- новые термины сначала проходят через статус `Кандидат` или явное решение о существующем термине;
- допустимые статусы терминов: `Кандидат`, `Принят`, `Заменён`, `Устарел`, `Запрещён`;
- нормативные слова требований BytePress: `ОБЯЗАН`, `ЗАПРЕЩЕНО`, `СЛЕДУЕТ`, `ДОПУСТИМО`, `МОЖЕТ`;
- нормативные слова требований применяются только в правилах, гейтах, процедурах, схемах, проверках и определениях требований;
- нормативные слова требований не используются как стилистическое украшение и не запускают массовую замену обычных слов в документах;
- отступление от `СЛЕДУЕТ` требует фиксации причины и риска в связанном плане, журнале, проверке или другом владельце прохода;
- английская форма термина допускается только как внешнее соответствие при первом упоминании, а в активных договорах основным именем остаётся русская форма принятого термина;
- терминологический зазор фиксируется, когда нужный смысл ещё не покрыт принятым термином;
- онтологический конфликт фиксируется, когда новый или изменяемый термин противоречит уже принятой модели сущностей и связей;
- решение по зазору или конфликту фиксируется закрытым значением: `Не_требуется`, `Принять`, `Заменить`, `Запретить`, `Оставить_кандидатом` или `Существующий_термин_достаточен`;
- создаваемый продукт получает только стартовый пакет терминов, а не полный словарь `BytePress`;
- устойчивые имена файлов, команд, веток, ID и технически неизбежные имена не переводятся.
