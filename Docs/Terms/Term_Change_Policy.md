# Term_Change_Policy

## Назначение
Этот документ определяет порядок ввода, проверки, замены и запрета терминов в BytePress.

## Порядок изменения
1. Проверить, существует ли уже термин с тем же смыслом.
2. Если конфликт найден, принять одно из решений:
   - принять существующий термин;
   - заменить новым термином существующий;
   - запретить новый термин.
3. Зафиксировать решение в карточке термина и связанных документах.
4. Обновить `Дата_изменения` и при необходимости связанные индексы.

## Запрещено
- вводить новый термин без проверки конфликта;
- дублировать определение термина в других документах;
- использовать свободные текстовые ссылки вместо ID термина в системных документах.
