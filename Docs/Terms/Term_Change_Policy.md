# Term_Change_Policy

## Назначение
Этот документ определяет порядок ввода, проверки, замены и запрета терминов в BytePress.

## Порядок изменения
1. Проверить, существует ли уже термин с тем же смыслом.
2. Если нужный смысл не покрыт принятым термином, зафиксировать терминологический зазор.
3. Если новый или изменяемый термин противоречит принятой модели сущностей и связей, зафиксировать онтологический конфликт.
4. Для нового термина использовать статус `Кандидат`, пока не принято решение.
5. Если зазор или конфликт найден, принять одно из закрытых решений:
   - `Существующий_термин_достаточен`;
   - `Принять`;
   - `Заменить`;
   - `Запретить`;
   - `Оставить_кандидатом`.
6. Зафиксировать решение в карточке термина и связанных документах.
7. Обновить `Дата_изменения` и при необходимости связанные индексы.

## Запрещено
- вводить новый термин без проверки конфликта;
- обходить статус `Кандидат` при появлении нового смысла без явного решения владельца;
- дублировать определение термина в других документах;
- использовать свободные текстовые ссылки вместо ID термина в системных документах.
