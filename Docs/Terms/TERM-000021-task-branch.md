# Рабочая ветка

ID: TERM-000021
Термин: Рабочая ветка
Статус: Принят
Связи: TERM-000009, TERM-000020, RULE-000016
Источник: Корректировка стартового контура после полевого теста
Дата_создания: 2026-04-24
Дата_изменения: 2026-05-01
Решение_по_конфликту: Принять как обязательный носитель прохода с правками

## Определение
Рабочая ветка формата `<type>/<NNNNNN>-<slug>`, из которой начинается любой проход с правками и через которую проходит локальная проверка, push и PR-путь.

Допустимые типы:
- `chore/`;
- `feature/`;
- `fix/`;
- `docs/`.

Для первого аналитического прохода продукта, созданного начальным развёртыванием, рекомендуемый тип — `chore/`.

`docs/` применяется только для обычных документационных проходов после снятия стартового гейта.

`product/` не используется как рекомендуемый или допустимый тип рабочей ветки.
