## Определение
Task-ветка формата `<type>/<NNNNNN>-<slug>`, из которой начинается любой writable pass и через которую проходит локальная проверка, push и PR-путь.

Допустимые типы:
- `chore/`;
- `feature/`;
- `fix/`;
- `docs/`.

Для первого аналитического прохода bootstrap-created продукта рекомендуемый тип — `chore/`.

`docs/` применяется только для обычных документационных проходов после снятия стартового гейта.

`product/` не используется как рекомендуемый или допустимый тип рабочей ветки.
