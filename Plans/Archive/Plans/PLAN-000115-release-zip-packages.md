## Шаги
1. Создать zip-пакеты.
   - Описание: Выполнить `git archive --format=zip --output=... <tag>` для `0.1.0`, `0.2.0` и `0.3.0`.
   - Определение_готовности: В `Plans/Archive/Releases/<version>/` есть соответствующий `BytePress-<version>.zip`.
2. Проверить zip-пакеты.
   - Описание: Выполнить `python3 -m zipfile -t` для каждого zip и сверить список файлов zip со списком `git ls-tree -r --name-only` соответствующего tag.
   - Определение_готовности: Проверки целостности zip пройдены, missing/extra равны `0`.
3. Обновить manifest и release archive README.
   - Описание: Зафиксировать путь, размер, проверку состава zip и границу, что zip не является текущим источником истины.
   - Определение_готовности: Manifest и README не спорят с фактическим состоянием release archive.
4. Закрыть плановый и журнальный контур.
   - Описание: Добавить `BACK-000127`, `PLAN-000115`, `CHG-000127`, `QL-000122` и закрыть `ROAD-000047`.
   - Определение_готовности: `ROAD-000047`, `BACK-000127` и `PLAN-000115` имеют статус `Завершено`, `ROAD-000048` не начат.
5. Выполнить проверки и подготовить PR.
   - Описание: Выполнить обязательные проверки, сделать commit, отправить ветку и создать PR в `develop` через `gh`.
   - Определение_готовности: Проверки пройдены, ветка отправлена, PR создан.

## Итог
`BACK-000127`, `PLAN-000115` и `ROAD-000047` завершены. Zip-пакеты исторических выпусков `0.1.0`, `0.2.0` и `0.3.0` созданы через `git archive` от tag, проверены через `python3 -m zipfile -t` и сверены с `git ls-tree -r --name-only` соответствующих tag.
