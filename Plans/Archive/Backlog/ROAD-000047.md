Дата_изменения: 2026-05-23

### Описание
Отделить архив завершённых планов от архива этапов, создать структуру release archive, описать назначение manifest и подготовить post-release очистку без потери проверяемости. Zip-архивы не создавать; если для них нужен отдельный manifest-договор, оставить это следующим шагом внутри `ROAD-000047`.

### Закрытие
Задача завершена в `PLAN-000112`. Завершённые планы перенесены из `Plans/Archive/` в `Plans/Archive/Plans/`, архив этапов сохранён в `Plans/Archive/Backlog/`, создан release archive `Plans/Archive/Releases/` и описан manifest-контур. Прямые ссылки на архивные планы синхронизированы. `ADR-000028` не создан, потому что новое архитектурное решение не принималось: проход материализовал уже утверждённый архивный контур. `bp_lint.py` затронут только минимальной синхронизацией двух hardcoded-путей после переноса архива; product bootstrap, zip-архивы и `ROAD-000048` не затронуты.

## BACK-000125 — Закрепить manifest-договор release archive и zip-пакета
ID: BACK-000125
Название: Закрепить manifest-договор release archive и zip-пакета
Тип: Стандарт
Приоритет: Высокий
Статус: Завершено
Связи: ROAD-000047, PLAN-000113, CHG-000125, QL-000120
Источник: Запрос владельца от 2026-05-24
Дата_создания: 2026-05-24
Дата_изменения: 2026-05-24

### Описание
Закрепить состав release manifest, связь manifest с logs, tag, release commit, release PR, post-release sync PR, `ROAD/BACK/PLAN` и проверками, а также условия допустимости zip-пакета. Zip-архивы и manifest конкретных версий не создавать; следующий проход должен заполнить manifest для релизов `0.1.0`, `0.2.0`, `0.3.0`.

### Закрытие
Задача завершена в `PLAN-000113`. `Plans/Archive/Releases/README.md` закрепил manifest как проверяемый мост между release package, `Logs/ReleaseLog.md`, `Logs/ChangeLog.md`, `Logs/QualityLog.md`, tag, release commit, release PR, post-release sync PR и `ROAD/BACK/PLAN`. Добавлен `Plans/Archive/Releases/MANIFEST_TEMPLATE.md`. Zip допустим только как проверяемый исторический package с manifest и проверкой состава; крупные бинарные zip-архивы не добавляются в репозиторий без отдельного решения владельца. Manifest конкретных версий и zip-файлы не создавались. `ADR-000028` не создан, потому что новое архитектурное решение не принималось: проход уточнил уже открытый архивный договор.

## Следующий шаг этапа
Zip-файлы создавать только при отдельном решении владельца и после проверки состава по manifest.
