# Plans

`Plans/` хранит плановый контур развития `BytePress`.

## Назначение
- держать дальний горизонт, активные задачи и один текущий план прохода;
- отделять планирование от журналов фактов и временного исполнения;
- обеспечивать связь `Roadmap -> Backlog -> Plan`.

## Границы
`Plans/` не хранит факты качества, решений или выпусков. Они фиксируются в `Logs/*`. Временные outputs инструментов не подменяют текущий `Plan`.

## Карта артефактов
- `Roadmap.md` — этапы системы и дальний горизонт.
- `Backlog.md` — активный реестр задач текущего этапа или явная фиксация отсутствия активного этапа.
- `Plans/PLAN-<NNNNNN>-<slug>.md` — текущий план одного прохода.
- `Plans/Archive/PLAN-<NNNNNN>-<slug>.md` — архив завершённых планов.
- `Plans/Archive/Backlog/ROAD-<NNNNNN>.md` — архив реестра задач завершённого этапа.

## Правила
- числовая часть ID содержит 6 знаков;
- текущий `Plan` описывает только один проход;
- в активном слое существует только один текущий `Plan`;
- `Backlog.md` хранит задачи только текущего этапа;
- если активного этапа нет, `Backlog.md` явно фиксирует пустое состояние;
- `Roadmap.md` не дублирует задачи реестра работ.
