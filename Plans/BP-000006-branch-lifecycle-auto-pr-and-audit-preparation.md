# PLAN-000006 — Branch lifecycle, Auto-PR process и подготовка к большому аудиту

ID: PLAN-000006
Название: Branch lifecycle, Auto-PR process и подготовка к большому аудиту
Статус: Завершено
Связи: BACK-0017, BACK-0018, BACK-0019, ADR-0013
Источник: Следующий проход после фиксации минимального Git/PR-контура
Дата_создания: 2026-03-14
Дата_изменения: 2026-03-17
Основание: После фиксации базового branch/PR-контура нужно довести жизненный цикл веток до полностью управляемого вида, определить рабочий Auto-PR процесс и подготовить вход для отдельного глубокого аудита системы.
Связанные_требования:
Связанные_backlog: BACK-0017, BACK-0018, BACK-0019
Связанные_ADR: ADR-0013

## Шаги
1. Уточнить branch lifecycle до рабочего регламента. DoD: для `develop`, task-веток, merge и удаления merged branch есть единый проверяемый порядок без повторного использования head-ветки.
2. Описать и проверить операционный контур Auto-PR после `push`. DoD: определены предусловия по `gh`, аутентификации, шаблону PR и fallback-команде при недоступности среды.
3. Подготовить вход в большой аудит системы. DoD: есть согласованный перечень зон аудита по naming, semver, `Plans/`, `Tools/`, дублям и связям без запуска рефакторинга в рамках этого плана.

## Риски
- целевой Auto-PR процесс может быть принят как уже реализованная автоматизация во всех средах;
- смешение подготовки к большому аудиту с самим рефакторингом сорвёт ограничение текущего контура;
- несогласованные правила удаления merged branch создадут повторное использование закрытых веток.
